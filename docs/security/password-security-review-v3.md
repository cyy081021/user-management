# 用户管理系统 — 密码安全审计报告 (v3)

## 密码策略

| 项目 | 要求 |
|------|------|
| 最小长度 | 12 位 |
| 大写字母 | ≥1 |
| 小写字母 | ≥1 |
| 数字 | ≥1 |
| 特殊字符 | ≥1 |
| 存储 | PBKDF2 哈希 (Werkzeug) |
| 来源 | 环境变量，源码无明文 |
| 占位拒绝 | "在此处填写"、placeholder 等 → 启动失败 |

## 启动校验流程

```
启动
 ├─ 读取 ADMIN_PASSWORD (环境变量)
 ├─ 读取 ALICE_PASSWORD   (环境变量)
 ├─ 检查是否为占位符/弱词 → 拒绝并退出
 ├─ 检查强度是否达标      → 拒绝并退出
 ├─ generate_password_hash() → USERS
 ├─ 生产模式强制 SECRET_KEY → 缺失则拒绝启动
 └─ 启动完成
```

## 认证流程

```
POST /login
 ├─ 读取当前 IP+用户名 维度延迟值（不递增计数）
 ├─ 防枚举基线延迟 (0.5~1.5s) + 渐进延迟等待
 ├─ check_password_hash() 比对
 │   ├─ 成功 → 清除失败计数 → 登录
 │   └─ 失败 → 按 IP+用户名 分桶记录失败次数
 │              渐进延迟阶梯：1s → 2s → 4s → 8s → 16s → 30s → 60s
 │              不向客户端暴露"锁定"状态，统一返回"用户名或密码错误"
 └─ 安全响应头 + CSRF + 速率限制
```
