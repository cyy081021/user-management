## 第 3 轮加固改进

### 11️⃣ 启动时拒绝占位密码

| 项目 | 内容 |
|------|------|
| **修复方式** | `_get_env_or_fail()` 检测包含 "在此处填写"、placeholder 等占位符 → 拒绝启动 |
| **涉及文件** | `app.py` |

---

### 12️⃣ 启动时强制校验密码强度

| 项目 | 内容 |
|------|------|
| **规则** | ≥12 位，含大写、小写、数字、特殊字符，不达标直接 `sys.exit(1)` |
| **涉及文件** | `app.py` |

---

### 13️⃣ 生产模式禁用 HTTP 公网暴露

| 项目 | 内容 |
|------|------|
| **修复方式** | `./run.sh prod` 仅绑定 `127.0.0.1`，对外需通过 Nginx 反向代理 |
| **涉及文件** | `run.sh` |

---

### 14️⃣ Session Cookie Secure 标志

| 项目 | 内容 |
|------|------|
| **修复方式** | `SESSION_COOKIE_SECURE = True`（HTTPS 模式下），防止 session cookie 经 HTTP 发送 |
| **涉及文件** | `app.py` |

---

### 15️⃣ SECRET_KEY 从环境变量读取

| 项目 | 内容 |
|------|------|
| **修复方式** | 优先读取 `SECRET_KEY` 环境变量，未设置时自动生成（并给出警告） |
| **涉及文件** | `app.py` |

---

### 16️⃣ 限流改用 Redis 共享存储

| 项目 | 内容 |
|------|------|
| **修复方式** | `storage_uri="redis://127.0.0.1:6379/0"`，Gunicorn 多 worker 共享计数 |
| **涉及文件** | `app.py` |

---

### 17️⃣ 按 IP+用户名维度渐进延迟替代硬锁定

| 项目 | 内容 |
|------|------|
| **规则** | 登录失败后按 IP+用户名 分桶记录，渐进延迟: 1s→2s→4s→8s→16s→30s→60s，不暴露"锁定"状态 |
| **涉及文件** | `app.py` |

---

### 18️⃣ 日志侧信道清理

| 项目 | 内容 |
|------|------|
| **修复方式** | 不记录具体密码内容，仅记录事件「用户 X 登录失败（第 N 次）」 |
| **涉及文件** | `app.py` |

---

### 19️⃣ 依赖清单 requirements.txt

| 项目 | 内容 |
|------|------|
| **修复方式** | 新增 `requirements.txt`，固定 Flask、Werkzeug、Flask-Limiter 等版本 |
| **涉及文件** | `requirements.txt`（新增） |

## 密码策略
